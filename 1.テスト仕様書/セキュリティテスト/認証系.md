## 1. パスワードなしでログインできないか

### ■ テスト目的

パスワード未入力で認証が通らないことを確認する。

### ■ 前提条件

- ログイン画面へアクセス可能であること

### ■ 手順

1. ユーザー ID のみ入力する
2. パスワードは空欄のままログインボタンを押下する

### ■ 期待結果

- エラーが表示される
- ログイン処理は行われない
- セッションは開始されない

---

## 2. ログイン試行回数制限があるか

### ■ テスト目的

一定回数ログイン失敗をするとロックがかかることを確認する。

### ■ 前提条件

- 試行回数制限が仕様として存在すること（例：5 回）

### ■ 手順

1. 誤ったパスワードを入力してログインを失敗させる（規定回数まで繰り返す）
2. ロック後に正しいパスワードでログインを試行する

### ■ 期待結果

- 規定回数失敗後にアカウントがロックされる
- ロック中は正しいパスワードでもログインできない
- 適切なエラーメッセージが表示される

---

## 3. セッション固定攻撃が効かないか

### ■ テスト目的

ログイン前後でセッション ID が適切に再生成されることを確認する。

### ■ 前提条件

- セッション ID が Cookie に設定される方式であること

### ■ 手順

1. ログイン画面へアクセスし、現在のセッション ID を取得する
2. ログイン処理を実施する
3. ログイン後のセッション ID を確認する

### ■ 期待結果

- ログイン前とログイン後でセッション ID が異なる
- セッション ID が固定されず、再生成されている

---

## 4. ログアウト後に戻るボタンで復活しないか

### ■ テスト目的

ログアウト後にブラウザの戻る操作で再度画面にアクセスできないことを確認する。

### ■ 前提条件

- ログインが成功している状態

### ■ 手順

1. 正常にログインする
2. 任意の認証必須ページへ遷移する
3. ログアウト処理を行う
4. ブラウザの戻るボタンを押す

### ■ 期待結果

- 認証が必要な画面が表示されない
- ログイン画面またはエラー画面へリダイレクトされる
- セッションが無効化されている
