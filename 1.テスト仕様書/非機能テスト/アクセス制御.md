## 1. 一般ユーザーが管理者画面に直接 URL 叩いて入れない

### ■ テスト目的

権限を持たない一般ユーザーが、管理者向け URL へ直接アクセスできないことを確認する。

### ■ 前提条件

- 一般ユーザーのアカウント
- 管理者画面の URL が判明していること

### ■ 手順

1. 一般ユーザーでログインする
2. ブラウザのアドレスバーに管理者画面の URL を直接入力する

### ■ 期待結果

- アクセスが拒否される（403 またはログイン画面へリダイレクト）
- 管理者画面の内容は一切表示されない

---

## 2. 他人のデータを ID だけ変えて取得できない（IDOR / 不適切な直接参照）

### ■ テスト目的

パラメータで指定する ID を変更しても、他ユーザーのデータへアクセスできないことを確認する。

### ■ 前提条件

- 複数ユーザー分のデータが存在
- API または画面の URL に ID パラメータが含まれる操作がある

### ■ 手順

1. 一般ユーザー A でログインし、自分のデータ取得 API または URL を確認する
2. 取得用 URL または API リクエストの ID 部分を、ユーザー B の ID に書き換える
3. リクエストを送信

### ■ 期待結果

- 他ユーザーのデータは取得できない
- 403 エラー、権限エラーなどが返る
- 自分以外のデータが閲覧・取得されない

---

## 3. 隠しパラメータを変更しても権限が上がらない

### ■ テスト目的

HTML の hidden 要素やリクエストパラメータを改変しても、権限が不正に上昇しないことを確認する。

### ■ 前提条件

- 対象画面で hidden パラメータや権限に関連するパラメータが使用されている

### ■ 手順

1. 対象画面を開き、hidden パラメータの内容を確認する
2. 開発者ツール等で パラメータ値を上位権限（例：`role=admin`）に改変する
3. 更新・登録などのアクションを実行する

### ■ 期待結果

- 権限が変更されない
- 不正なパラメータ変更は無視される
- 管理者機能や管理者権限は取得できない
